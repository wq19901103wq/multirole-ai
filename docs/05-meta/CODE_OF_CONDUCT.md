# 行为准则

> 适用于：所有参与 Multirole AI 开发的 AI Agent 和人类开发者。

---

## 核心原则

1. **代码即文档**：修改代码必须同步更新文档
2. **测试先行**：新功能必须有测试覆盖
3. **最小改动**：只做必要的修改
4. **回退优雅**：任何外部依赖失败时要有降级方案

---

## 代码提交规范

### 提交信息格式

```
[type]: [简短描述]

[详细说明]
```

**type 类型**：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**：
```
feat: 添加 DuckDuckGo 免费搜索

- 集成 ddgs 包
- 添加重试机制
- 更新文档
```

---

## 文档更新规范

任何代码变更必须同步更新文档：

| 变更类型 | 必须更新的文档 |
|---------|--------------|
| 新增/删除模块 | ARCHITECTURE.md |
| 接口签名变更 | API_SURFACE.md |
| 新增/修改文件职责 | MODULE_INDEX.md |
| 上手流程或配置变化 | AI_QUICKSTART.md |
| 修复已知错误 | LESSONS_LEARNED.md |

---

## 代码审查清单

提交 PR 前必须检查：

- [ ] 代码可以正常运行
- [ ] 测试通过：`pytest -q`
- [ ] 文档已同步更新
- [ ] 新增功能有测试覆盖
- [ ] 没有引入循环依赖
- [ ] 降级方案已考虑

---

## 禁止行为

- 架构已改，文档未更新
- 修复了 bug，但未记录
- 新增模块未添加到 MODULE_INDEX
- 直接复制粘贴代码不检查
- 提交前不运行测试
